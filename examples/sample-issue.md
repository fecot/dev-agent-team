# Sample Issue: ユーザー一覧画面にメールアドレス検索を追加する

実際のワークフロー出力例は [`sample-workflow-output.md`](sample-workflow-output.md) を参照してください。

## 必須

### 背景・目的

- なぜこの変更が必要か: CSチームから「特定ユーザーを探すのに時間がかかる」という声が複数寄せられている。現在のユーザー一覧画面はページネーションのみ対応しており、特定ユーザーを探すには目視で何ページもめくる必要がある。問い合わせ対応中にアカウント特定に時間がかかり、対応時間が長引く要因になっている
- 期待する効果: 問い合わせメールに記載されたアドレスを貼り付けて、該当ユーザーを即座に特定できるようになり、CS対応時間が短縮される

### スコープ

- やること:
  - ユーザー一覧画面の上部にメールアドレス検索フォームを追加する
  - 部分一致で検索できる（`@example.com` のようなドメイン部分での絞り込みも可能）
  - 大文字小文字を区別しない
  - 入力をクリアすると元の一覧に戻る
  - 該当ユーザーが0件のときは、空状態メッセージを表示する
- やらないこと:
  - 名前・電話番号など他フィールドの検索（検索対象は **メールアドレスのみ**）
  - 検索条件の保存・履歴機能
  - 既存のページネーション挙動の変更

### 受け入れ基準

- [ ] 検索フォームに任意のメールアドレス文字列を入力すると、一致するユーザーのみ表示される
- [ ] 大文字小文字に依存せず一致する
- [ ] 0件のとき「該当するユーザーが見つかりません」相当の空状態が表示される
- [ ] 入力をクリアするとフィルタが解除される
- [ ] 既存のページネーション・他のソート機能と併用しても破綻しない
