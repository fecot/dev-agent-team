# Skill: 影響範囲分析

## 概要

コード変更の波及範囲を特定するスキル。ArchitectureReviewer エージェントが使用する。

## 分析の観点

### コードの依存関係
- 変更対象の関数・クラスを使っているコード
- 変更対象が依存しているコード（双方向を確認）
- 共有ユーティリティ・ヘルパーへの影響

### データへの影響
- DBスキーマの変更が必要か
- 既存データの整合性に影響するか
- マイグレーション手順が必要か

### API・インターフェースへの影響
- レスポンス形式の変更
- 引数の変更（後方互換があるか）
- 外部クライアントへの影響

### テストへの影響
- 変更対象をテストしているファイル
- モック・スタブの修正が必要か

### 非機能要件への影響
- パフォーマンス（ループ内の処理追加、N+1）
- セキュリティ（認証・認可の変更）
- ログ・モニタリングへの影響

## 深刻度の基準

| 深刻度 | 基準 |
|-------|------|
| 高 | 本番データに影響 / 認証・認可に関わる / 外部APIへの影響 |
| 中 | 既存機能の振る舞いが変わる / テストの修正が必要 |
| 低 | 新規追加のみ / 内部リファクタリング |
