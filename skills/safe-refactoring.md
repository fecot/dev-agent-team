# Skill: 安全なリファクタリング

> 主に Phase 5（Safe Implementation）で使用する。

## 概要

既存の振る舞いを変えずにコードを改善するスキル。ImplementationDriver エージェントが使用する。

## 原則

### 1. リファクタリングと機能追加を混ぜない
1つのPRにリファクタリングと機能追加を同時に含めない。
先にリファクタリングを入れ、通過したら機能追加する。

### 2. テストがない箇所をリファクタリングしない
テストなしでリファクタリングすると、壊れたことに気づけない。
先にテストを書いてからリファクタリングする（テストファースト）。

### 3. 小さい差分で進める
1ステップ = 1意味の変更。
大きなリファクタリングは複数のPRに分割する。

### 4. 命名の変更は慎重に
命名変更は全参照箇所の更新を伴う。
IDEのリネーム機能を使い、手動置換を避ける。

## よくある安全でないリファクタリング

- 動作確認なしにロジックを抽出する
- ついでに不要コードを削除する（影響範囲未確認）
- コメントアウトされたコードを削除する（削除理由不明）
- 「わかりやすく書き直す」と称して処理を変える

## チェックリスト

- [ ] 変更前後でテストが同じように通る
- [ ] 変更の意図をコミットメッセージで説明できる
- [ ] 機能変更を含んでいない
